Notification System

Real-time notifications triggered by request status changes, comments, and overdue reminders. Includes 24-hour deduplication cooldown and read/unread tracking via 30-second polling.

24-Hour Deduplication: Before creating, checks: Notification.objects.filter(recipient=user, request=req, type=type, created_at__gte=now()-24h).exists(). If a duplicate exists within 24 hours, the notification is silently skipped. This prevents spam when status changes happen rapidly.
Sub-Process 4.2
Poll & Display Notifications
Frontend polls the server every 30 seconds to fetch unread notifications. Displayed as a badge counter in the header and a dropdown list.

D4: Notifications
Polling Interval: 30 seconds via setInterval() in useNotifications() hook. Cleanup on component unmount.
Why not WebSockets? Polling was chosen over WebSocket for simplicity — the 6-person team scale doesn't justify the added complexity of WebSocket infrastructure, connection management, and reconnection logic.
Display: Bell icon in header shows unread count badge. Clicking opens a dropdown with notification list sorted by newest first.
