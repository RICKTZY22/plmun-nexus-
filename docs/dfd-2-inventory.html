Input (15 fields): name, category (ELECTRONICS/FURNITURE/EQUIPMENT/SUPPLIES/OTHER), quantity, status (AVAILABLE/IN_USE/MAINTENANCE/RETIRED), location, description, image (file), access_level (STUDENT/FACULTY/STAFF/ADMIN), priority (LOW/MEDIUM/HIGH), is_returnable (bool), borrow_duration (int), borrow_duration_unit (HOURS/DAYS/WEEKS)
Default Values from Settings: Staff can set default category, status, and location in Settings → Inventory tab. These pre-fill the form to speed up bulk item entry.
Image Upload: Sent as multipart/form-data. Stored in /media/items/ directory. URL returned as relative path.
Permission: IsStaffOrAbove — returns 403 Forbidden for Student and Faculty roles.

Sub-Process 2.4
Change Item Status
Transitions an item between AVAILABLE, IN_USE, MAINTENANCE, and RETIRED states. Requires a note explaining the change. Maintenance status accepts an ETA date.
🛡️ Staff / Admin
new_status + note

From Status Allowed Transitions Additional Data
AVAILABLE MAINTENANCE, RETIRED status_note required; maintenance_eta for MAINTENANCE
IN_USE AVAILABLE (Returned) Quick return without modal (handleQuickReturn)
MAINTENANCE AVAILABLE (Fixed) Clears maintenance_eta and status_note
RETIRED AVAILABLE (Reactivated) Restores item to active inventory
Audit: Logs "Status Change" with old_status → new_status and the note
Sub-Process 2.5
Browse & Filter Inventory
All authenticated users can browse inventory. Results are filtered server-side by access_level and client-side by search term, category, and status.
👤 All Users
search, category, status
2.5
Browse
Inventory
Filtered items array
D2: Items
Filter Parameter Example
Text Search ?search= ?search=laptop → name__icontains="laptop"
Category ?category= ?category=ELECTRONICS → exact match
Status ?status= ?status=AVAILABLE → exact match
Access Level (automatic) Student sees STUDENT items; Staff sees all items
Views: Card grid (default on mobile) or data table (desktop). Configurable in Settings → Appearance.
Grouping: Items grouped by stock level: Out of Stock → Low Stock (≤5) → Normal Stock. Each group is collapsible.
Pagination: Client-side, configurable 6/12/24/48 items per page.
