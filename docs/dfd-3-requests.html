Sub-Process 3.1
Submit Borrow Request
Any authenticated user creates a request to borrow an item. The item name is denormalized (snapshot stored on the request) to survive future item renames or deletions.

Sub-Process 3.3
Approve Request & Deduct Stock
Staff approves a request. Stock is deducted atomically using Django F() expressions to prevent race conditions when two staff approve the same request simultaneously.
🛡️ Staff
Approve action

3.3
Approve
status → APPROVED
D3: Requests
Atomic Deduction (Critical):
rows = Item.objects.filter(id=item.id, quantity__gte=qty).update(quantity=F('quantity') - qty)
If rows == 0 → insufficient stock → reject with "Not enough stock available".
This prevents double-deduction when two staff click "Approve" at the same moment.
Fields Updated: request.status="APPROVED", request.approved_by=staff_user, request.approved_at=now()
Notification: Requester receives "Your request has been approved" notification
