Sub-Process 1.5
Refresh Access Token
When the 60-minute access token expires, the client sends the refresh token to get a new access token without re-entering credentials. Includes a mutex queue to handle concurrent refreshes.
🌐 Client Browser
refresh_token
1.5

🌐 Client Browser
Endpoint: POST /api/auth/token/refresh/ with { refresh: "eyJ..." }
Mutex Queue: If 3 API calls fail with 401 simultaneously, only 1 refresh request is made. The other 2 wait in a queue and retry with the new token once refresh completes.
Failure: If refresh token is also expired (>7 days), user is logged out and redirected to /login
